The Online Reviewer System is a comprehensive platform that facilitates a review-based earning model where users purchase products through mediators, submit reviews on e-commerce platforms, and receive refunds (typically 90% of the purchase amount). The system manages the entire lifecycle from order placement to refund completion.

### 2.2 Complete User Journey Map #### Journey 1: New User Onboarding

1. User discovers platform through mediator
2. Creates account (email, password, name)
3. Adds e-commerce platform accounts (Amazon, Flipkart, etc.)
4. Adds mediator details (name, WhatsApp)
5. Views dashboard with empty state
6. Ready to place first order
#### Journey 2: Order Placement Flow

1. User receives deal from mediator (via WhatsApp)
2. Opens app â†’ Clicks "New Order"
3. Fills order details:
   - Selects platform (Amazon/Flipkart/etc.)
   - Selects account (auto-filtered by platform)
   - Enters product details
   - Enters amounts (order & expected refund)
   - Assigns mediator
4. Places actual order on e-commerce platform
5. Updates order status to "Ordered"
6. System generates notifications for tracking
#### Journey 3: Order Fulfillment Flow

1. Product delivered â†’ Updates status to "Delivered"
2. User reviews product on platform â†’ Updates to "Deliverables Done"
3. User submits refund form â†’ Updates to "Refund Form Done"
4. System tracks refund timeline
5. If delayed â†’ Auto-updates to "Overdue Passed for Refund Form"
6. User follows up with mediator â†’ "Remind Mediator for Payment"
7. Receives refund â†’ Updates to "Refunded"
8. Order cycle complete
#### Journey 4: Monitoring & Management

Daily Activities:
- Check dashboard for pending actions
- Review notifications for due dates
- Update order statuses
- Message mediators for follow-ups
- Export data for records

Weekly Activities:
- Review earnings summary
- Analyze mediator performance
- Clean up completed orders
- Plan upcoming orders

## 6. System Calculations & Business Logic (All in INR â‚¹) ### 6.1 Financial Calculations

All amounts are in Indian Rupees (â‚¹)
Storage: Integer in paise (1 rupee = 100 paise)
Display: â‚¹X,XX,XXX format

Profit per Order = Refund Amount (â‚¹) - Order Amount (â‚¹)
Success Rate = (Refunded Orders / Total Orders) Ã— 100
Average Profit Margin = Average(All Profits) / Average(Order Amounts) Ã— 100
Pending Investment = Sum(Order Amounts where Status != Refunded/Cancelled) in â‚¹
Expected Returns = Sum(Refund Amounts where Status != Refunded/Cancelled) in â‚¹
### 6.2 Time-Based Calculations

Delivery Time = Delivery Date - Order Date
Review Submission Time = Refund Form Date - Delivery Date
Payment Processing Time = Refunded Date - Refund Form Date
Total Cycle Time = Refunded Date - Order Date
### 6.3 Mediator Performance Metrics

Active Orders = Count(Orders where Mediator = X AND Status != Refunded/Cancelled)
Total Investment = Sum(Order Amounts for Mediator X) in â‚¹
Success Rate = (Refunded Orders / Total Orders) for Mediator X
Average Processing Time = Average(Cycle Time) for Mediator X

Past Delivery Date: Warning but allow (for historical data)
Refund > Order Amount: Warning but allow (special cases)
### 14.2 Data Validation Rules
Email: Valid format with @ and domain
Phone: 10 digits with country code
Amounts: Positive integers only
Dates: Logical sequence (order < delivery < refund)
URLs: Valid HTTP/HTTPS format
